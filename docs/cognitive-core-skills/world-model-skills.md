# World Model Skills

World model skills represent the structure and dynamics of environments so systems can predict state transitions and evaluate alternatives.

## Scope

- Physical world modeling
- Digital workflow modeling
- Enterprise process modeling
- Social world modeling
- Causal dynamics
- Temporal dynamics
- Object permanence
- State transition prediction
- Simulation
- Counterfactual environment prediction
- Outcome forecasting
- Real-time environment tracking

## Why this matters

Without world modeling, systems often produce reactive, local decisions. With world modeling, systems can forecast downstream effects, pre-empt failures, and choose lower-risk plans.

## Operational use patterns

| Pattern | Example |
|---|---|
| Transition prediction | Predict queue state after changing routing policy |
| Simulation rollout | Compare two staffing plans before execution |
| Counterfactual testing | Estimate outcome if escalation happened earlier |
| Forecasting | Predict SLA and throughput under planned changes |

## Integration points

- Planning: better dependency and contingency design
- Reasoning: stronger causal and counterfactual judgments
- Verification: reduced surprise through expected-state comparison
- Governance: clearer risk forecasts and escalation triggers
